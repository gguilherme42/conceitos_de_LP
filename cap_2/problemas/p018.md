## As linguagens continuam a evoluir. Que tipo de restrições você acha adequada para mudanças em linguagens de programação? Compare suas respostas com a evolução de Fortran.

As restrições que pensam que são adequadas para mudanças são:
 - Não existir vetores dinâmicos
 - O usuário não pode definir seus tipos
 - Não é possível trablhar com outros paradigmas além do da linguagem
 - Não possuir o paradigma imperativo

Evolução do Fortran:
 - Formatação de entrada e saída
 - Sub-rotinas definidas pelo usuário
 - Sentenças de seleção
 - Sentenças de repetição
 - Variáveis com nomes maiores
 - Compilação independente de sub-rotinas
 - Passar subprograma como parâmetros para outros sub-programas
 - Adição de tipos explícitas 
 - If com cláusula else
 - Sentenças de controle laços lógicos
 - Vetores dinâmicos, registros, sentença de seleção múltipla e os módulos
 - Posições específicas para sentenças foi abandonada

Alguns pontos de evolução são iguais aos meus.
